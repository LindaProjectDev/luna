# Security Policy

## Supported Versions
Le développement actif est sur `dev`. Les releases stables sont sur `main`.

## Reporting a Vulnerability
Merci de ne pas ouvrir d’issue publique pour une vulnérabilité.
Ouvre une discussion privée ou contacte le mainteneur via GitHub.

Inclure :
- description du problème
- impact
- étapes de reproduction
- version / commit concerné
